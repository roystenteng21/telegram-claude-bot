## Coding Standards (locked)
- **R1:** Route by cost — exact match → regex → keyword → cached lookup → live data → Claude. Never call Claude for routing decisions.
- **R2:** Single pass, immediate exit. Once a handler matches, execution stops. No fallback re-runs detectors. Loops exit on first match.
- **R3:** One sheet read per request, at the latest possible moment. Cached in memory, invalidated only on write. No sheet read during routing.
- **R4:** Haiku for classification/extraction/JSON under 200 tokens. Sonnet for reasoning/conversation/multi-step only. Set min max_tokens.
- **R5:** Typing indicator before every external API call. Parallel calls use asyncio.gather(). Nothing blocks the event loop.
- **R6:** Session messages never reach main routing chain. Session data is flat dict with explicit fields. Sessions time out cleanly.
- **R7:** Cache hierarchy: in-memory → cached sheet read → live sheet read → external API. Each layer reached only if above misses.
- **R8:** Every branch sets reply or returns. Empty reply is a bug. Claude fallback for genuine unknown intent only.
- **R9:** After every file edit, verify with `python3 -c "import <module>"` — not just `ast.parse`. This catches NameErrors, missing class definitions, and module-level execution failures that syntax checks miss.
- **R10:** After any str_replace touching a class or function boundary, view the surrounding lines of the output file before proceeding to the next edit.

## Architecture Notes
- **Routing:** Primary elif chain runs once. No else block re-runs detectors. Session handlers exit before reaching main router.
- **Caches:** \_merchant\_cache, \_card\_names\_cache, \_system\_prompt\_cache. Invalidate at write site only. Warm card cache on startup.
- **Models:** Haiku: is\_calendar\_request, parse\_expense\_text\_v2, classification, parse\_bill\_request, parse\_overseas\_request. Sonnet: conversation fallback, reasoning, market narrative.
- **Sheets:** setup\_sheets() uses single worksheets() call. No repeated API reads in setup. Em Log and Dev Notes never read in routing.
- **Module Layers:** Import order strictly enforced (no reverse imports ever): config → clients → state → sheets → helpers → feature modules → sessions → routing → infrastructure → bot.py. Feature modules never import each other except meetings→crm (one-way). routing.py is the only file that imports everything.
- **Module Registry:** Module Registry tab in the Em Management Sheet is the live index of all modules. Columns: Module · File · Layer · Key Functions · Imports From · Last Changed · Session · Status. Updated automatically by deploy.py on every deploy.
- **Bills:** As of S26, bills store a full due date (DD/MM/YYYY), not a day-of-month integer. send\_bill\_reminders compares against specific dates. check\_missed\_items\_on\_startup parses full date strings.
- **Birthday pending:** As of S26, birthday\_pending is never wiped by the 2pm followup job. Only cleared on explicit sent/skip reply or at midnight by auto\_clear\_birthday\_pending. Skip and midnight clear both write "not sent" to CRM.
- **Birthday style reference:** Pasted birthday messages stored to Settings sheet under key `birthday_message_style`. generate\_birthday\_greeting reads this at call time for style matching.
- **Trip setup:** As of S27, trip setup is single-message parse via Haiku — no multi-step wizard. Destination and dates are critical fields; currency is auto-derived from destination. Overseas mode activates immediately on parse. If critical fields are missing, Em asks once (all missing fields in one message). `\_trip\_setup` is only used for the one follow-up state (`step: awaiting\_missing`). `\_trip\_setup` is expired on boot if older than 6 hours or missing `started\_at` timestamp (via `\_expire\_stale\_trip\_setup\_inline` called from `load\_trip\_setup\_from\_sheet`). `looks\_like\_new\_intent` silently clears `\_trip\_setup` mid-session. Cancel/nevermind/stop/abort exits trip setup at any point.
- **Todos:** As of S27, completing a todo replies with struck task (MarkdownV2 ~strikethrough~) + remaining pending list in one message. Struck task does not persist after that message.
- **Session expiry:** receipt\_confirm\_sessions age check on restore is implemented but commented out (debug flag) — enable once confirmed stable.

## Rules
- **Ship Rule:** Nothing ships until fully wired, tested, and deployed in the same session.
- **Feasibility Rule:** Never suggest a solution without certainty it is feasible — applies to code, claims, and performance predictions alike.
- **No Silent Changes:** Any change to module boundaries, import structure, coding standards, or deploy flow must be logged to Dev Notes immediately.
- **Circular Import Zero Tolerance:** Never work around a circular import. Restructure instead.
- **Em Log Rule:** Every session must produce a complete Em Log entry. Built/Fixed/Pending must be specific.
- **Cross-Feature Rule:** Only permitted cross-feature import: meetings.py → crm.find\_row (one-way). All other cross-feature calls go through routing.py dispatch. No feature module calls another feature module's handler functions.
