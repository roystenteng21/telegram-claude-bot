## Architecture
21-module flat architecture. Import order (strict, no reverse): config → clients → state → sheets → helpers → feature modules → sessions → handlers → routing → infrastructure → bot.py. Only permitted cross-feature import: meetings→crm (one-way). routing.py is sole omnimporter. handlers.py owns all session handler logic (Layer 5.5).
**Repo:** roystenteng21/telegram-claude-bot (public) · **Runtime:** Railway · **Deploy:** deploy.py

---

## Coding Standards
R1: Route by cost — exact→regex→keyword→cache→live→Claude. Never Claude for routing.
R2: Single pass, immediate exit on first match.
R3: One sheet read per request, cached, invalidated on write only.
R4: Haiku for classification/extraction <200 tokens. Sonnet for reasoning/conversation only.
R5: ALL iCloud CalDAV calls must use asyncio.to_thread() — blocking network calls freeze the event loop on Railway. Apply to any external blocking call.
R6: Session messages never reach main routing chain. Sessions time out cleanly.
R7: Cache hierarchy: in-memory → cached sheet → live sheet → external API.
R8: Every branch sets reply or returns. Empty reply is a bug.
R9: Verify with `python3 -c "import <module>"` after edits, not just ast.parse.
R10: View surrounding lines after any str_replace touching class/function boundaries.
R11: Every except block must send a user-facing Telegram message — never swallow silently.
R12: Every sheet field read must be cast to str() before string operations.
R13: Error messages must never use parse_mode="Markdown" — send via plain reply_text only.
R14: deploy.py auto-detects all .py module files — never hardcode module lists.

## Key Architecture Decisions
- **Bills:** Full due date DD/MM/YYYY stored (not day integer). S26+.
- **Birthday pending:** Never wiped by 2pm job. Only cleared on explicit sent/skip or midnight.
- **Trip setup:** Single-message Haiku parse. Stale \_trip\_setup expired on boot (>6h).
- **Todos:** Complete → struck task + remaining list in one message (MarkdownV2).
- **Sessions age check:** receipt\_confirm\_sessions drops sessions >6h old on restore. Enabled S29. Re-confirmed active S31.
- **Receipt uploads:** Personal OAuth2 via personal\_drive\_service (clients.py). Service account has no storage quota on personal Drive. state.OAUTH\_DRIVE\_OK flag set on boot. Boot alert if broken. reauth.py refreshes token.
- **OAuth token:** GOOGLE\_OAUTH\_TOKEN env var in Railway (base64 token.json). client\_secrets.json and token.json are gitignored — never commit.
- **Receipt flow:** Photo → upload Em Receipts/YYYY-MM/DD-MM-YYYY-merchant.jpg → vision parse → confirmation ALWAYS shown (yes / save only / split / edit) → log on yes.
- **force_confirm:** Photos always pass force_confirm=True to handle_expense_text — bypasses auto-log even for known merchants. Implemented S31.
- **alert_error:** Shared async helper in helpers.py — sends Telegram message on silent failures in critical paths. Uses state._app_ref. Never raises. Also fires on Claude fallback to alert routing gaps.
- **handlers.py:** New Layer 5.5 module. Owns all session handler logic extracted from routing.py. routing.py delegates to it via await calls. Never import routing.py from handlers.py.
- **Calendar trigger:** `cal [title] [calendar] [date] [time] [location]` — parses via Haiku with pre-computed date anchors, shows confirmation, writes on yes. `cal delete [title]` — finds upcoming event, confirms before delete.
- **Calendar sessions:** calendar_confirm_sessions in state.py. Inline edit supported (e.g. "change time to 2pm"). All iCloud ops async via asyncio.to_thread().
- **Calendar date resolution:** Relative dates (next Friday, tomorrow, etc.) pre-computed in Python and injected into Haiku prompt — never left to model inference. Year always injected explicitly.
- **iCloud CalDAV on Railway:** UNRESOLVED — CalDAV connections hang indefinitely from Railway's network despite asyncio.to_thread() and 10s timeout. Local connections work fine. Root cause unknown — may be Railway blocking outbound CalDAV or iCloud rate-limiting Railway IPs. Needs investigation next session.
- **Routing fallback alert:** Claude fallback now fires alert_error with the message that fell through — visible in Telegram for routing gap detection.
- **GitHub fetch:** All module files fetched via `https://raw.githubusercontent.com/roystenteng21/telegram-claude-bot/main/<file>.py`. Em Codes Drive folder retired.
